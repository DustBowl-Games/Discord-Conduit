SINGLETON WARNING
-----------------
This deployment runs exactly ONE replica on purpose. The Discord gateway
connection is a singleton: a second running pod would double-handle every slash
command. Do NOT scale this deployment above 1.

VIEW LOGS
---------
  kubectl logs -f -l app.kubernetes.io/instance={{ .Release.Name }}

CHECK STATUS
------------
  kubectl get pods -l app.kubernetes.io/instance={{ .Release.Name }}

There is no Service or Ingress: the bot has no inbound ports. It is an outbound
gateway/REST client only.
